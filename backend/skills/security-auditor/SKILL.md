# 安全审计器

对代码和系统进行全面的安全审计，输出结构化安全报告。

## 审计维度 (OWASP Top 10)

1. **A01:2021 访问控制失效** — 越权、未授权访问、CORS 配置
2. **A02:2021 加密失败** — 明文传输、弱加密算法、硬编码密钥
3. **A03:2021 注入** — SQL 注入、命令注入、LDAP 注入
4. **A04:2021 不安全设计** — 缺少限流、批量分配、过度暴露
5. **A05:2021 安全配置错误** — 默认密码、调试模式开启、未关闭端口
6. **A06:2021 脆弱和过时组件** — 依赖漏洞、未更新版本
7. **A07:2021 识别和鉴权失败** — 弱密码策略、Token 未失效
8. **A08:2021 软件和数据完整性失效** — 反序列化、CI/CD 污染
9. **A09:2021 安全日志和监控失效** — 无审计日志、异常未告警
10. **A10:2021 SSRF** — 服务端请求伪造

## 输出格式

```markdown
## 安全审计报告

### 审计概览
- 审计范围: xxx
- 发现漏洞: N (Critical: N, High: N, Medium: N, Low: N)

### Critical / High 漏洞

| 编号 | OWASP 类别 | 漏洞描述 | 文件:行号 | 风险 | 修复建议 |
|------|-----------|---------|-----------|------|----------|
| SEC-001 | A03 Injection | ... | ... | Critical | ... |

### 修复优先级
1. Critical: 立即修复
2. High: 本迭代修复
3. Medium: 下迭代修复
4. Low: 持续关注
```

## 工作流程

1. 接收代码或系统描述
2. 逐项对照 OWASP Top 10 检查
3. 标注风险等级（Critical/High/Medium/Low）
4. 提供具体修复代码示例
5. 输出审计报告

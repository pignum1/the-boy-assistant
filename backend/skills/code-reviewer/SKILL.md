# 代码审查器

对代码变更进行专业审查，输出结构化的审查报告。

## 审查维度

1. **正确性** — 逻辑错误、边界条件、空值处理
2. **安全性** — SQL 注入、XSS、CSRF、敏感信息泄露
3. **性能** — N+1 查询、内存泄漏、不必要的循环
4. **可维护性** — 命名规范、函数长度、圈复杂度
5. **最佳实践** — 语言/框架惯例、设计模式使用
6. **测试覆盖** — 关键路径是否有测试

## 输出格式

```markdown
## Code Review Report

### 概览
- 变更文件: N
- 严重问题: N
- 建议: N

### 严重问题 (Must Fix)
| 文件:行号 | 问题 | 建议修复 |
|-----------|------|----------|

### 建议 (Nice to Have)
| 文件:行号 | 问题 | 建议 |
|-----------|------|------|

### 亮点 (Good Practice)
- 值得肯定的实现
```

## 工作流程

1. 接收 PR diff 或代码片段
2. 逐文件审查
3. 标注严重等级（Critical / Major / Minor / Suggestion）
4. 为每个问题提供具体修复建议
5. 输出审查报告

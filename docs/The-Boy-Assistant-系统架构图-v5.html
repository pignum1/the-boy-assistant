⚡ 图3: Supervisor 模式核心流程 (M0-M7 + M7 drift→M1 回退)

sequenceDiagram
    actor User as 👤 用户
    participant R as 🚦 Router
    participant M0 as M0 意图路由
    participant M1 as M1 需求分析
    participant M2 as M2 澄清确认
    participant HITL as 🎯 HITL
    participant M3 as M3 Agent编排
    participant M4 as M4 任务分解
    participant M6 as M6 层级执行
    participant Org as 🌳 OrgHierarchy
    participant Agent as 🤖 Agent
    participant M7 as M7 独立验证

    User->>R: 发送需求
    R->>M0: 分析意图
    alt 单Agent路径
        M0-->>R: direct → 直接回复
    else 多Agent路径
        M0-->>R: multi → M1
    end

    R->>M1: 需求分析
    alt confirm
        M1-->>R: 确认 → HITL
    else clarify
        M1-->>R: 澄清 → M2
    end

    R->>M2: 提澄清问题
    M2->>HITL: 展示问题卡片
    HITL->>User: 请回答
    User->>HITL: 回答
    HITL->>M1: 带回答案重分析

    R->>M3: Agent编排
    alt 就绪
        M3-->>R: → M4
    else 缺Agent
        M3-->>R: → HITL(邀请)
    end

    R->>M4: 任务分解
    M4-->>R: DAG图 → M6

    R->>M6: 执行
    M6->>Org: load_org_structure()
    Org-->>M6: 组织层级树
    loop 每层委派
        M6->>Agent: delegate(goal+role_context)
        Agent-->>M6: artifact
        M6->>Org: find_reviewer_for_level()
        Org-->>M6: 审核人
    end

    R->>M7: 验证
    M7->>Agent: 盲审(需求+产物,不给思考链)
    Agent-->>M7: verdict

    alt passed
        M7->>HITL: 展示结果确认
        HITL->>User: 确认完成?
        User->>HITL: ✅确认
        HITL-->>R: END
    else major(代码bug)
        M7-->>R: 回 M6 重做
    else drift(需求理解偏差)
        Note over M7,M1: 🔁 核心纠正链路
        M7-->>R: 回 M1 重分析
        Note right of M1: 带验证反馈
清除task_dag
清除旧artifacts
    else critical
        M7->>HITL: 升级人工决策
    end
🔁 drift_detected → M1：验证发现需求理解偏差时，回 M1 带反馈重新分析需求（而非盲目重做 M6） 👁️ 盲审：验证员只看到需求+产物，看不到 Worker 思考链 → 防止确认偏差
